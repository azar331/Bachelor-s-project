# Tbilisi Public Transport Route Optimizer

მოცემული პროექტი წარმოადგენს თბილისის საზოგადოებრივი ტრანსპორტის ქსელის ოპტიმიზაციის პროგრამულ მოდელს, რომელიც შეიქმნა კომპიუტერული მეცნიერებების მიმართულების საბაკალავრო ნაშრომის ფარგლებში. სისტემა იყენებს გრაფთა თეორიასა და მოდიფიცირებულ დეიკსტრას (Dijkstra) ალგორითმს.

ტრადიციული სანავიგაციო სისტემებისგან განსხვავებით, ეს ალგორითმი ითვალისწინებს ტრანსპორტის მოძრაობის გრაფიკს, ინტერვალებსა და ავტობუსის ლოდინის დროს. ამის საფუძველზე, პროგრამა ადგენს რეალურად ყველაზე სწრაფ მარშრუტს დროის კონკრეტული მომენტისთვის.

## ძირითადი მახასიათებლები

დინამიური წონები: ალგორითმი ითვლის არა მხოლოდ მგზავრობის ფიზიკურ დროს, არამედ გადაჯდომისას შემდეგი ავტობუსის ლოდინის დაყოვნებასაც.

ინტელექტუალური საძიებო სისტემა: მომხმარებელს არ მოეთხოვება გაჩერების ზუსტი ID-ის ან ქუჩის ნომრის ცოდნა. ქუჩის სახელის შეყვანისას, პროგრამა std::map სტრუქტურის გამოყენებით თავად უკავშირებს მას უახლოეს ობიექტებსა და გაჩერებებს.

მაღალი წარმადობა: გამოთვლები სრულდება 1 წამზე ნაკლებ დროში. უმოკლესი გზის ძიების ალგორითმის დროითი სირთულეა O((V + E) log V).

## მონაცემთა სტრუქტურა

პროგრამა ლოკალური გრაფის ასაგებად იყენებს სამ ძირითად CSV ფაილს. სისტემის გამართულად მუშაობისთვის აუცილებელია, რომ სამივე ფაილი იმყოფებოდეს იმავე დირექტორიაში, სადაც შესრულებადი ფაილია:

1. stop_graph.csv - ინახავს გრაფის წიბოებს. მასში მოცემულია ორი მეზობელი გაჩერების ID და მათ შორის მგზავრობის დრო.
2. bus_stop_info.csv - წარმოადგენს ქუჩების დასახელებების, გეოგრაფიული ობიექტებისა და გაჩერებების ID-ების რუკას.
3. bus_routes.csv - ინახავს ინფორმაციას თითოეული ავტობუსის ხაზის შესახებ (ავტობუსის ID და გაჩერებების თანმიმდევრობა).

## ტექნოლოგიური სტეკი

პროექტი დაწერილია C++ ენაზე (ISO/IEC 14882:2020 - C++20 სტანდარტი). მეხსიერების ეფექტური მართვისა და მონაცემთა სწრაფი დამუშავებისთვის ფართოდ არის გამოყენებული სტანდარტული ბიბლიოთეკის (STL) სტრუქტურები.

## ინსტალაცია და კომპილაცია

პროექტის გასაშვებად საჭიროა C++ კომპილატორი (მაგალითად GCC ან Clang).

1. მოახდინეთ რეპოზიტორიის კლონირება:
git clone https://github.com/azar331/Bachelor-s-project
cd რეპოზიტორიის-სახელი

2. გაუშვით კომპილაციის ბრძანება ტერმინალში (რეკომენდებულია C++20 სტანდარტი):
g++ -std=c++20 main.cpp -o transport_optimizer

3. გაუშვით შესრულებადი ფაილი (Linux/macOS):
./transport_optimizer

Windows ოპერაციულ სისტემაზე გაუშვით:
transport_optimizer.exe

პროგრამის გაშვების შემდეგ, ტერმინალი მოგთხოვთ შემდეგი პარამეტრების ეტაპობრივად შეყვანას:

* ავტობუსების ინტერვალი (Period) - საშუალო ინტერვალი, რომლითაც მოძრაობენ ავტობუსები.
* საწყისი ლოკაცია - შეიყვანეთ მიმდინარე ქუჩის დასახელება. პროგრამის მიერ შემოთავაზებული სიიდან აირჩიეთ სასურველი გაჩერების იდენტიფიკატორი.
* დანიშნულების ადგილი - შეიყვანეთ საბოლოო დანიშნულების ქუჩის დასახელება და აირჩიეთ გაჩერება.
* მიმდინარე დრო - შეიყვანეთ მიმდინარე დრო წუთებში.

ალგორითმი დააგენერირებს და დაბეჭდავს უმოკლეს მარშრუტს, გამოსაყენებელი ავტობუსების ნომრებს, გადაჯდომის წერტილებსა და მგზავრობის ჯამურ დროს.
